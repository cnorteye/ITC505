Addendum

The process of creating the Lights Out game involved several phases, each contributing to a final, interactive game that is both functional and engaging.

For the HTML Structure, I created a basic structure with a div for the game board and sections for move count and timer. In the CSS Styling, I used grid layout for the game board to arrange the squares effectively. For the JavaScript Functionality, I implemented a game logic including square toggling, move counting and timer functionality. I also added the ability to reset the game.

I made a few creative choices that I believe enhanced the website. I chose bright colors for the game squares to make the toggling effect visually appealing. For example, I choose the default yellow color because it stands out against the game’s background. I also integrated a timer and move counter to enhance the competitive aspect of the game. The timer adds a sense of urgency, while the move counter helps players track their efficiency. To add an interesting element to the Lights Out game page, I used a customer Google fonts for the Font Serving feature. I included the Google Fonts stylesheet link the the HTML head section. In the CSS file, I applied the custom font 'MooLahLah-Regular' font family to the body of the page. I also added the 'rubik-pixels-regular' and 'rubik-maze-regular' to the paragraph tags in the addendum to add some visual marks in the text.

Like with any website design, there were some challenges that came up along the way. One of the challenges I faced was ensuring that the game state resets correctly upon refresh or using the reset button. I overcame that by implementing a resetBoard function that clears the board and resets the timer and move count. Another issue I encounted was synchronizing the timer with the game’s start and reset functionality. I was able to resolve that by creating functions to start and update the timer, ensuring that it starts with the game and stops when the game is won. I also added an alert to the user once the game is won. Lastly, I managed the timer state using JavaScript intervals and handled resetting the timer appropriately. Last, but not the least, there was a challenge with ensuring that user input (clicking on squares) is correctly processed and that adjacent squares are toggled as expected. My solution for that was to implemented logic to toggle squares and their adjacent ones, with checks for boundaries to prevent errors. I used event listeners effectively to handle user interactions.

One of my takeaways from creating this game is to that regular testing is essential for refining game mechanics and interface design. Iterative development allows for the identification and resolution of issues early in the process. Additionally, it is also important to balance visual appeal with functionality which will add to enhancing the overall user experience. For example, choosing appropriate colors, providing clear instructions, and ensuring smooth interactions contribute to a more enjoyable game.
